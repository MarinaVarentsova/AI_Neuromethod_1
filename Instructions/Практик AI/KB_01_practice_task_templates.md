
# KB_01_practice_task_templates

## Назначение
Файл содержит шаблоны практических заданий.

## Структура практического задания
Для каждого задания указывай:
- название;
- цель задания;
- какую рабочую ситуацию моделирует;
- что должен сделать обучающийся;
- вводные данные;
- ограничения;
- рабочий артефакт на выходе;
- критерии оценки;
- типичные ошибки;
- подсказки для проверки;
- уровень сложности;
- связь с KPI или рабочим результатом.

## Кейс
Кейс подходит, если нужно проверить анализ ситуации, выбор решения, работу с ошибками и объяснение логики.

Структура:
| Элемент | Описание |
|---|---|
| Ситуация | Что происходит |
| Роль обучающегося | Кто действует |
| Задача | Что нужно сделать |
| Вводные | Какие данные есть |
| Ограничения | Что усложняет ситуацию |
| Решение | Что должен предложить обучающийся |
| Артефакт | Что он сдаёт |
| Критерии | Как оценивается результат |

## Рабочее задание
Рабочее задание подходит, если результатом должен стать артефакт: письмо, план, отчёт, скрипт, таблица, промпт, чек-лист, решение по кейсу.

## Симуляция
Симуляция подходит, если нужно воспроизвести процесс, диалог, цепочку решений или работу в системе.

## Role-play
Role-play подходит для коммуникации, переговоров, обратной связи, продаж, управления, поддержки клиентов и сложных разговоров.

## Branching scenario
Branching scenario подходит, если важно показать последствия выбора. Каждая развилка должна менять ход ситуации или проявлять ошибку мышления.
